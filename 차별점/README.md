# 차별점

3년차 프론트엔드 개발자로서 쌓아온 것들을 정리합니다.

---

## 숫자로 말하는 개선 이력

대부분의 개발자는 기능을 구현합니다.  
저는 구현한 것을 측정하고, 개선하고, 수치로 기록합니다.

| 프로젝트 | 개선 항목 | Before | After |
|---------|----------|--------|-------|
| KISTEP | Lighthouse 점수 | 45 | 95 |
| KISTEP | 배포 시간 | 5분 | 30초 |
| KISTEP | 개발 서버 로딩 | 1분 | 1초 |
| 보일러플레이트 | 신규 프로젝트 배포 | 5분 | 30초 |
| LLM 템플릿 | 신규 LLM 프로젝트 도입 | 1달 | 2주 |
| Cesium 3D 지도 | JS Heap 크기 | 2.45 MB | 0.35 MB (−86%) |
| Cesium 3D 지도 | 3D Tiles 스타일 조건 수 | ~5,000개 | ~20개 (−99%) |

## 혼자만 쓰지 않는 코드

개인 생산성이 아닌 팀 생산성을 고민해왔습니다.

[SimpleScaffold](https://github.com/SimpleScaffold) 는 여러 회사 프로젝트를 거치며 반복되는 설정을 체계화한 결과물입니다.  
Frontend(Vite/Next), Micro Frontend, Nginx, DB, 보안 툴킷, 디자인 시스템까지  
스택의 모든 레이어를 조직 단위로 관리하고 있습니다.

단순 코드 모음이 아니라 기획문서(ss-plan-doc-md)까지 포함된 구조가 이 사람의 시야를 보여줍니다.

---

## 전공이 코드에 녹아있다

정보보안학과 졸업 + K-Shield Jr. 수료.  
보안을 "나중에 추가하는 것"이 아니라 처음부터 설계에 반영합니다.

- 인증·인가 시스템 공통화 직접 설계
- 심층망 암호화 기반 키 관리 플랫폼(SecuAI) 구축 참여
- 배포 서버 보안 환경 설계
- 논문 게재: *A Study on the Risk and Countermeasures of Hacking Cable* (JKSCI, 2023)

프론트엔드 개발자 중 논문을 보안 저널에 게재한 케이스는 흔하지 않습니다.

## 9개 도메인, 일관된 품질

MLOps · CCTV 관제 · AI 챗봇 · 3D 지도 · 네트워크 시각화 · 인사관리 · 연구지원 · 예산회계 · 키 관리.  
도메인마다 요구하는 데이터 구조와 UX가 완전히 다릅니다.  
그럼에도 매 프로젝트에서 공통 컴포넌트 구조화, 성능 최적화, 배포 자동화를 일관되게 적용했습니다.

## 군에서도 수상한 사람

개발 이전의 경험도 다릅니다.  
보안 감사에서 여단장상을 받았고, 이는 육군 최초 스마트폰 시범운용 부대 선정으로 이어졌습니다.  
문제를 발견하고 해결책을 제시하는 태도는 직군이나 환경과 무관하게 유지되어 왔습니다.
