## SimpleScaffold — 조직 단위 보일러플레이트

https://github.com/SimpleScaffold

팀에서 반복되는 프로젝트 설정과 구조를 체계화한 보일러플레이트 모음입니다. 단순한 템플릿이 아니라, 팀이 어떤 프로젝트를 시작하든 일관된 품질과 빠른 배포가 가능하도록 전 레이어를 커버하는 것이 목표였습니다.

React(Vite), React(Next.js), Micro Frontend(Host·Remote·Shell)로 세 가지 프론트엔드 구조를 제공하고, Nginx 미디어 서버, DB Docker Compose, 보안 툴킷, KRDS 기반 디자인 시스템, 기획 문서 템플릿까지 포함합니다. 이 보일러플레이트의 도입으로 팀의 평균 배포 시간이 5분에서 30초로 줄었습니다.

Micro Frontend 구성은 Module Federation 기반으로 Host·Remote·Shell의 역할을 명확히 분리했습니다. 공통 UI와 유틸리티는 패키지 레이어(@repo/packages/fe-ui, fe-utils)로 분리해 여러 앱이 공유하도록 구성했고, 설치부터 마이크로앱 아키텍처, AI 에이전트 연동, 인증 설계까지 14개 섹션의 문서 체계로 정리했습니다.

**스택** React, Next.js, TypeScript, Vite, Module Federation, Tailwind CSS, shadcn/ui, Redux Toolkit, Docker, Nginx, GitHub Actions

## 온품 3D 지도 플랫폼 — Cesium 성능 최적화

디지털 트윈 기반 3D 공간 데이터 시각화 플랫폼입니다. 대용량 좌표 데이터와 3D Tiles를 다루면서 렌더링 성능 문제가 발생했고, 이를 측정·분석·개선·검증 사이클로 해결했습니다.

건물별 조건을 개별 생성하던 방식을 높이·층수 기반 구간 분류로 전환해 3D Tiles 스타일 조건을 5,000개에서 약 20개로 99% 줄였습니다. POI 가시성 계산을 매 프레임(postRender)에서 이동 종료 이벤트(camera.moveEnd)로 전환해 이동 중 CPU 부하를 0으로 낮췄고, React 컴포넌트 언마운트 시 Cesium 리소스를 정리하도록 생명주기를 연결했습니다. 최종적으로 JS Heap 메모리가 2.45MB에서 0.35MB로 86% 감소했습니다.

**스택** React, TypeScript, Cesium, Vite

## KISTEP 통합연구지원시스템 — 전방위 성능 개선

과학기술 연구 지원을 위한 대규모 정보 시스템으로, 기존 시스템의 성능 문제와 느린 개발 환경을 개선하는 작업을 담당했습니다.

코드 스플리팅과 번들 최적화로 Lighthouse 성능 점수를 45에서 95로 끌어올렸고, 배포 프로세스를 자동화해 배포 시간을 5분에서 30초로, Vite 기반 개발 서버 마이그레이션으로 로딩 시간을 1분에서 1초로 단축했습니다. D3.js로 다차원 네트워크 그래프 시각화를 구현하고, ElasticSearch 기반 고급 검색 기능도 담당했습니다.

**스택** React, TypeScript, D3.js, Django, Docker, ElasticSearch

## LLM 시각화 공통 템플릿 (LimeLab)

사내 LLM 프로젝트마다 UI를 처음부터 만드는 비효율을 없애기 위해 구축한 공통 템플릿입니다. 스트리밍 응답 처리, 채팅 인터페이스, 데이터 시각화 컴포넌트를 재사용 가능한 구조로 설계했고, Storybook으로 컴포넌트를 문서화해 디자인 시스템으로 발전시켰습니다. 이 템플릿 도입 후 새로운 LLM 프로젝트의 초기 도입 기간이 1달에서 2주로 단축됐습니다.

**스택** React, TypeScript, Storybook, Django, MariaDB
